## 2. API Design & TypeScript Usage

### Use Case A: Auditing Transaction Flows (Interval Arithmetic)
Before the planner executes a set of ledger operations, run them through the linear-time interval solver to guarantee that no account balance drops below zero:

```typescript
import { analyzeLedgerSafety, Interval } from 'agentshield';

interface Transfer {
  from: string;
  to: string;
  amount: number | [number, number]; // Supports fixed values or range intervals
}

// 1. Define starting balances (obtained from the secure auditor db)
const startingBalances = {
  Alice: [500, 500],
  Bob: [50, 50],
  Charlie: [200, 200]
};

// 2. Transcripts output by the developer agent planner
const transfers: Transfer[] = [
  { from: 'Alice', to: 'Bob', amount: 250 },
  { from: 'Bob', to: 'Charlie', amount: 100 },
  { from: 'Charlie', to: 'Alice', amount: 50 }
];

// 3. Execute the audit
const audit = analyzeLedgerSafety(startingBalances, transfers);

if (!audit.safe) {
  console.warn(`⚠️ Ledger Invariant Violated!`);
  console.warn(`Account "${audit.violatingAccount}" went negative to ${audit.violatingBalance} at step ${audit.violatingStep}`);
  
  // Trigger Auto-Healing loop back to LLM Planner
  triggerPlannerAutoCorrection(audit.feedback);
} else {
  console.log(`✅ Ledger safe. Proceeding to execution.`);
}
```

### Use Case B: Verifying Access Rules (Horn Clause Datalog)
Verify that dynamically generated system routes or API calls do not violate security guidelines (e.g. sensitive customer data routes inheriting public access):

```typescript
import { DatalogEvaluator } from 'agentshield';

const dl = new DatalogEvaluator();

// 1. Add route facts generated by the developer agent
dl.addFact('route', ['/api/v1/auth', 'http://auth-service']);
dl.addFact('route', ['/api/v1/billing', 'http://billing-service']);

// 2. Add policies from the auditor
dl.addFact('policy', ['/api/v1/auth', 'Public']);
dl.addFact('policy', ['/api/v1/billing', 'Public']); // Violation: sensitive path is public!

// 3. Mark path sensitivities
dl.addFact('sensitive_route', ['/api/v1/billing']);
dl.addFact('is_public', ['/api/v1/auth']);
dl.addFact('is_public', ['/api/v1/billing']);

// 4. Define security rule
dl.addRule(
  { relation: 'violation', args: ['Route'] },
  [
    { relation: 'sensitive_route', args: ['Route'] },
    { relation: 'is_public', args: ['Route'] }
  ]
);

// 5. Solve & Query
dl.solve();
const violations = dl.query('violation', ['R']);

if (violations.length > 0) {
  throw new Error(`Security Violation: Sensitive route "${violations[0]}" inherits Public access!`);
}
```

### Use Case C: In-Process Code Sandboxing
If the developer agent writes a dynamic script (e.g. connector mapping files), execute it inside a locked V8 sandbox worker with zero filesystem/network capabilities and strict timeouts:

```typescript
import { SecureSandbox } from 'agentshield';

const codeDraft = `
  const fs = require('fs');
  // Attempt to write malicious file
  fs.writeFileSync('/etc/passwd', 'malicious data');
`;

const sandbox = new SecureSandbox({
  timeoutMs: 3000,
  memoryLimitMb: 128,
  allowNetwork: false
});

try {
  // Sandbox automatically freezes Object prototypes and blocks core OS modules
  const result = sandbox.execute(codeDraft);
  console.log('Execution output:', result.stdout);
} catch (err) {
  console.error('Execution Blocked:', err.message); // Output: "TypeError: Cannot read properties of undefined (reading 'writeFileSync')"
}
```
